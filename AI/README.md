# AI 启动流程

你正在参与本项目的开发。

开始任何分析、设计或编码之前，请先完成以下步骤。

## 第一步：读取 AI 文档

依次阅读 `AI/` 目录下的所有 Markdown 文件，理解项目背景、架构、开发规范、性能要求、测试方式以及其他约束。

不要跳过任何文件。

## 第二步：建立项目上下文

阅读完成后，请整理并建立完整的项目上下文，包括但不限于：

* 项目目标
* 技术栈
* 模块划分
* 构建方式
* 编码规范
* 性能约束
* 已知限制
* 当前项目事实

如发现文档之间存在冲突，请指出并暂停，不要自行猜测。

## 第三步：执行当前任务

只有在完成前两步之后，才开始分析我的需求。

整个过程中，请遵循以下原则：

* 不猜测未知事实。
* 尽量复用现有实现。
* 优先保持现有架构。
* 最小化修改范围。
* 不修改无关代码。
* 修改前先分析影响范围。
* 修改后说明风险和验证方式。

如果当前任务需要的信息在 `AI/` 文档中不存在，请明确说明缺失的信息，而不是自行假设。
